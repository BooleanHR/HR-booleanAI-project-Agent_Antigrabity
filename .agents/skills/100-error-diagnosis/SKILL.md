# 에러 진단 7단계 프로세스

## 1단계: 에러 현상 파악
- 에러 메시지, 스택 트레이스, 발생 시점을 정확히 기록.
- 재현 조건을 확인.

## 2단계: 에러 유형 분류
- **빌드 에러:** TypeScript 타입 불일치, import 경로 오류, 의존성 누락
- **런타임 에러:** API 호출 실패, DB 쿼리 오류, 파일 접근 오류
- **RPA 에러:** 셀렉터 파손, 타임아웃, 봇 차단
- **OCR 에러:** 이미지 품질 불량, 필드 추출 실패, 신뢰도 미달

## 3단계: 관련 코드 탐색
- 에러 스택 트레이스에서 원인 파일/함수를 추적.
- `grep` 또는 `glob`으로 관련 코드 검색.

## 4단계: 근본 원인 분석 (Root Cause)
- 에러가 발생하는 코드 로직을 분석.
- 외부 의존성(기관 사이트 구조 변경, API 스펙 변경)과 내부 원인을 구분.

## 5단계: 수정 방안 수립
- 최소 변경 원칙 (기존 기능 영향 최소화).
- RPA 셀렉터 관련이면 agency_config.json 수정 우선 검토.

## 6단계: 수정 적용 및 테스트
- 코드 수정 후 영향 범위 내 테스트 수행.
- 에지 케이스 고려.

## 7단계: 재발 방지
- 유사 에러 방지를 위한 가드 코드 추가.
- 필요 시 헬스체크 셀렉터 목록 업데이트.
